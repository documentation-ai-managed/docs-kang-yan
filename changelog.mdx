<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

- Introduced real-time order status notifications via WebSockets.

- Enhanced mobile responsiveness for tablet ordering interfaces.

- Fixed order duplication issue during peak hours.
- Resolved payment gateway timeout errors for Alipay integration.

<Update label="2024-08-10" description="v1.0.0" tags={["feature", "breaking"]}>

## New Features

- Core ordering platform with menu builder and POS integration.
- User authentication via OAuth and API keys.

## Breaking Changes

- Updated API base URL from `/v0` to `/v1`.
- Deprecated legacy `order/create` endpoint; migrate to `orders/create`.

## Initial Setup

- Basic dashboard for restaurant owners.
- Customer-facing ordering interface.

</Update>

## Upgrade Process

Follow these steps to upgrade your Kang Yan integration safely.

<Steps>
  <Step title="Backup Data" icon="database">
    Export your current orders and menus.

    ```
    curl -X GET https://api.example.com/v1/export \
      -H "Authorization: Bearer YOUR_TOKEN"
    ```
  </Step>
  <Step title="Update SDK" icon="download">
    <Tabs>
      <Tab title="npm" icon="package">
````bash
npm update kang-yan-sdk@latest
````
      </Tab>
      <Tab title="Python" icon="python">
````bash
pip install --upgrade kang-yan-sdk
````
      </Tab>
    </Tabs>
  </Step>
  <Step title="Test Changes" icon="test-tube">
    Verify new search endpoint.

    <CodeGroup tabs="JavaScript,Python">
```javascript
const response = await fetch('https://api.example.com/v1/search?query=dimsum&category=rice', {
  headers: { Authorization: `Bearer ${YOUR_TOKEN}` }
});
```
```python
import requests
response = requests.get('https://api.example.com/v1/search', 
                       params={'query': 'dimsum', 'category': 'rice'},
                       headers={'Authorization': f'Bearer {YOUR_TOKEN}'})
```
    </CodeGroup>
  </Step>
</Steps>
